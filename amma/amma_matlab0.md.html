For the moment, I suggest you install the following: - Matlab (obviously) - Symbolic Math Toolbox - Statistics and Machine Learning Toolbox It is possible to add other toolboxes later on, if they are required. # Reviewing MATLAB basics with an example To help you refamiliarise yourself with Matlab, we will run through a simple Matlab workflow that involves loading a data set from an Excel (.xlsx) file into your local Matlab environment, and analysing that data set using in-built Matlab functions for summarising and visualising those data sets. This year, as much as possible, we will aim to solve problems using programming, and so you will be discouraged from using methods that rely on the Graphical User Interface (GUI), and you will be encouraged to write scripts, functions that you can use, develop and re-use throughout the module. ## The Matlab Environment But first, let's make sure we set up a Matlab Integrated Development Environment (IDE) that is appropriate for our needs in this module. Most of the time, it should look like figure [frMatlabIDE] below: ![Figure [frMatlabIDE]: Matlab Integrated Development Environment](images/matlab_ide_setup.png) The most important components of the MATLAB IDE are the Editor, the Command Window, the Workspace and the Current Folder section. If you aren't able to see any of these sections, Editor : This is the MATLAB text editor where scripts and functions[^other_editors] are written. Command Window : In-built MATLAB functions and user-built scripts or functions can be directly called from the command window. Some useful commands are given below. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ Python linenumbers % Calling a function or script (could be user-built *.m file) >> * % Accessing help for an in-built Matlab function * >> help * % Print working directory >> pwd % Clear workspace >> clear % Clear command window >> clc ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ Workspace : When variables are created, they show up in the Workspace. You can explore the properties of those variables from within the Workspace. Current Folder : Self-explanatory, shows the files and folders in your current working directory. Sometimes, you may want to access resources (e.g. data files, functions) that are not in the current folder or in any of the folder in the MATLAB path. In that case, you should be able to identify where the relative path of the resources in relation to the current working directory.
